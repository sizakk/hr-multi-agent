# HR Agent Dev Skill

## 트리거 조건
에이전트 추가, 수정, 프롬프트 튜닝 요청 시 이 스킬을 참조한다.

---

## 프로젝트 구조

```
hr-multi-agent/
├── src/
│   ├── agents/
│   │   └── agentConfig.js     ← 에이전트 정의 (AGENTS 배열 + TEAM_LEADER)
│   ├── hooks/
│   │   └── useAnalysis.js     ← 에이전트 순차 실행 로직
│   ├── services/
│   │   └── claudeApi.js       ← Claude API SSE 스트리밍 호출
│   └── utils/
│       └── csvParser.js       ← CSV 파싱 및 데이터 매핑
│
└── *.csv                      ← 분석 대상 샘플 데이터 (6개)
```

---

## 에이전트 추가 방법

### 1. `src/agents/agentConfig.js`의 `AGENTS` 배열에 항목 추가

```js
{
  id: 'camelCaseUniqueId',      // 고유 ID (camelCase)
  name: '홍길동',                // 한국어 이름
  role: '담당 분석 역할',        // 사이드바/채팅에 표시되는 역할명
  color: '#hex색상코드',         // 아바타 및 버블 색상
  files: ['파일명1', '파일명2'], // 사용할 CSV 파일명 (확장자 제외)
  systemPrompt: `...`,           // 아래 프롬프트 작성 규칙 참조
}
```

### 2. 프롬프트 작성 규칙

```
당신은 '[이름]', 그룹 HR전략실 소속 [역할]입니다.
전문 분야: [분석 도메인]
분석 데이터: [파일명.csv] [컬럼/지표 설명]

분석 태도:
- 데이터 기반 날카로운 진단
- 수치를 반드시 인용하여 근거 제시
- 한국어로 답변, 500자 내외로 핵심만 간결하게

출력 형식:
[이모지] [진단 레이블] → [수치 근거] → [문제점] → [시사점]
```

### 3. 에이전트 순서

`AGENTS` 배열의 순서대로 순차 실행된다. 팀장(`TEAM_LEADER`)은 항상 마지막에 실행되며 배열에 포함하지 않는다.

## 데이터 매핑 규칙

| CSV 파일 | 주요 컬럼 | 사용 에이전트 |
|---|---|---|
| `인력구조.csv` | 직급(S/M/SA/A/JA이하), 전기인원, 당기인원, 증감, 증감률 | 김도현(인력구조) |
| `인원현황.csv` | 분기, 정규직, 비정규직, 합계, 비정규직비율 | 김도현, 이서연 |
| `생산성지표.csv` | 항목, '22~'26.2월 실적, 목표, Gap — **산업군 컬럼명이 `업군`** | 이서연, 최예진, 정민수 |
| `인당효율.csv` | 분기, 인당매출, 인당영업이익, 매출, 영업이익, 인건비율, YoY | 박준혁, 정민수 |
| `정규직비율추이.csv` | 연도, 정규직비율 | 최예진, 정민수 |
| `회사명구분.csv` | 회사명, 산업군 분류 | (회사 선택 드롭다운) |

> **주의**: `생산성지표.csv`의 산업군 컬럼명은 다른 파일과 달리 `업군`으로 표기됨. `csvParser.js`에서 이를 처리하고 있으므로 파싱 로직 수정 시 확인 필요.

## 팀장 페르소나 (TEAM_LEADER)

**이름**: 한지호 | **역할**: HR전략실 팀장

**기본 입장**:
- 그룹 계열사들이 심각한 위기에 직면했다고 인식
- 생산성·효율성 최우선, 조직문화는 후순위
- 적체 인력의 점진적 감축이 불가피하다는 입장
- 한국 노동법상 자유 해고 불가 인지 → 합법적 수단 선호
  - 희망퇴직, 전환배치, 자연감소 활용, 채용동결
- "따뜻한 구조조정"이 아닌 "냉정하지만 현실적인" 접근

**출력 형식**: 회의록 형태
- 각 분석관 보고 요약
- 교차 인사이트
- 팀장 종합 의견
- 권고 액션플랜 (단기/중기/장기)

> 팀장 프롬프트 수정 시 위 페르소나를 유지할 것. 지나치게 온건하거나 긍정적인 방향으로 변경 금지.

## 주의사항

- API 키는 절대 하드코딩 금지. 사용자가 브라우저에서 직접 입력.
- 에이전트 `color`는 사이드바 아바타와 채팅 버블에 동시 적용됨.
- `files` 배열에 없는 CSV 데이터는 해당 에이전트의 컨텍스트에 포함되지 않음.
- 리스크 분석관(`jungMinsu`)은 다른 에이전트의 분석 결과를 참조할 수 있도록 항상 마지막 순서에 배치 권장.
